Complete Functional Requirement Specification

Employee Portal

Employees enter ID, name, department, designation, mobile, email, optional blood group, and optional joining date. The page supports mobile camera capture and existing image upload, validates the photo, auto-crops to a portrait ID-card ratio, and shows a final confirmation screen before submission.

Photo Rules

The processor checks image resolution, portrait suitability, blur score, browser face detection availability, face count, and crop confidence. The output is JPG, default 300 x 400 px, 300 DPI target, 35mm x 45mm reference size.

Admin Portal

Admins search submissions, edit details, approve, reject with remarks, request resubmission, preview photos, export employee data, download approved photos, download department-wise packages, and generate a print-ready ID-card data package.

Security

Production deployment should use authenticated roles for employee, reviewer, admin, and auditor; object storage signed URLs; encrypted image storage; immutable audit logs; rate limiting; and server-side image validation before approval.

User Flow Diagrams

Employee opens PWA->Reads photo guidelines->Enters details->Captures/uploads photo
Smart validation->Auto crop + resize->Preview->Submit pending approval
Admin login->Review queue->Approve or reject->Export print package

Responsive UI/UX, Admin, Export, PWA, and Source Structure

UI/UX Designs

Mobile-first single-column employee form, camera preview with face guide, compact print settings, processed preview, and fixed bottom-friendly action buttons. Desktop expands to two-column review and admin dashboards.

Face Detection Logic

Use the browser FaceDetector API where available. Server production should use OpenCV, RetinaFace, MediaPipe, or a managed vision API to enforce one-face validation consistently across browsers.

Export and Printing

Approved photos are exported as JPG files named by employee ID. The print package includes CSV metadata, manifest JSON, audit status, and department folders for direct handoff to ID-card printing software.

PWA Approach

Cache the app shell, use camera APIs over HTTPS, keep drafts in IndexedDB/localStorage, add background sync for server deployments, and store images in object storage after upload.
